# 100 Days of DevOps – KodeKloud Challenge

This repository documents my hands-on journey through the **KodeKloud 100 Days of DevOps challenge**, where I solve real-world infrastructure and system administration tasks daily and record them in a structured, production-style format.

The goal is to build strong practical skills in Linux, networking, automation, and service deployment while maintaining professional documentation similar to real operational runbooks.

---

## 📋 Overview

This repository documents a structured, hands-on progression through the **KodeKloud 100 Days of DevOps challenge**, covering the end-to-end lifecycle of modern infrastructure and application delivery.

The challenge begins with foundational Linux system administration and security, then expands into version control workflows, containerization with Docker, orchestration with Kubernetes, continuous integration and delivery using Jenkins, configuration management with Ansible, and infrastructure provisioning using Terraform.

Each task simulates scenarios commonly encountered in enterprise environments, such as securing remote access, deploying and scaling web applications, managing distributed services, automating operational workflows, and defining infrastructure as code. The documentation reflects not only the commands used, but also the operational reasoning behind configuration choices and troubleshooting steps.

---

## 🛠️ How Each Day Is Documented

Each day’s work is documented using a consistent operational format so that every task can be easily reviewed, reproduced, and understood in context.

Every daily README includes:

* **Task Overview** – Describes the scenario and objective
* **Requirements** – Lists constraints and system details
* **Implementation Steps** – Step-by-step commands and configuration
* **Verification** – Commands used to confirm successful deployment

This structure reflects real-world **runbooks and change documentation** used in production DevOps environments.

## 🎯 Objectives of This Challenge

* Strengthen Linux system administration skills
* Gain practical experience with common infrastructure services

## 🧰 Technologies & Tools Covered

This challenge progresses through a layered DevOps toolchain, starting from operating system fundamentals and moving toward containerization, orchestration, CI/CD, and infrastructure automation.

Core areas covered in this repository include:

* **Linux Administration:** User management, permissions, services, process troubleshooting, and system security
* **Networking & Security:** SSH, firewalls, SELinux, SSL/TLS configuration
* **Version Control:** Git workflows including branching, rebasing, hooks, and pull request management
* **Containerization:** Docker image creation, container networking, volumes, and multi-container applications with Docker Compose
* **Orchestration:** Kubernetes deployments, services, volumes, secrets, rolling updates, and troubleshooting distributed workloads
* **CI/CD:** Jenkins server setup, pipeline configuration, scheduled jobs, and secure build environments
* **Configuration Management:** Ansible inventory, playbooks, modules, templating, and service orchestration
* **Infrastructure as Code:** Terraform for provisioning and managing infrastructure resources
